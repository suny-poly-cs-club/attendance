# CS Club Attendance App Deployment

## Deployment steps for the attendance app

Requirements: Docker, Docker Compose

1. Clone this repo or download the compose file: [docker-compose.yml](/docker-compose.yml)
2. Create a `.env` file and set it up according to [.env.example](/.env.example)
3. Start everything using `docker compose -d`

## Using HTTPS
By default the attendance system runs on http only. If you would like to run it with https and have certificates generated by LetsEncrypt, replace any related `docker compose` commands with:
```sh
$ docker compose -f docker-compose.yml -f docker-compose.https.yml up -d
```

### Managing a Deployment
> [!TIP]
> If you get a permission error when running the Docker commands or scripts included in this repository, either add your user to the `docker` group or run them again with `sudo`.

**Promoting a user:** Users can either be promoted to service admin through the Users tab of the UI, or with the [promote](/promote) script (`./promote <email>`)

## Updating
The Docker images are automatically built on push in CI. To update the running installation, use:
```sh
$ docker compose pull
$ docker compose up -d
```

This repository also includes submodules for the frontend and backend repositories, should you want to view the source code. To update them to the lastest versions, use:
```sh
$ git submodule update --init --remote --recursive
```
